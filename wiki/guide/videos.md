# 非 JAV 视频管理

除了以番号为中心的 JAV 影片体系，SakuraMedia 也支持纳管**没有番号、也没有外部元数据**的普通视频——比如个人收藏、自己录制或剪辑、国产 / 国外资源等。这套体系和 JAV 影片完全平行，定位是「**仅播放 + 整理**」，不掺和订阅、下载这类 JAV 专属的自动化能力。

## 它和 JAV 影片有什么不同

| 维度 | JAV 影片 | 非 JAV 视频 |
|---|---|---|
| 核心标识 | 番号 | 无番号，标题默认取文件名 |
| 元数据来源 | JavDB / DMM 等外部抓取 | 无外部元数据，全部手动维护 |
| 组织维度 | 女优、标签、合集 | **人物**、标签、**视频合集** |
| 自动化能力 | 订阅、自动下载、推荐、相似影片、以图搜图 | **都不提供** |
| 播放能力 | 缩略图、进度、时刻、流播放 | **完全复用同一套** |
| 片段收藏 | 支持 | 同样支持（见[视频片段收藏](/guide/clips)）|

简单说：非 JAV 视频借用了 SakuraMedia 的整套播放底座（缩略图、播放进度、精彩时刻、流播放），但不参与任何依赖番号 / 外部元数据的自动化链路。

## 怎么组织非 JAV 视频

- **标签**：复用和影片同一套通用标签体系，方便统一检索。
- **人物**：非 JAV 视频用独立的「人物」维度，**不复用 JAV 的女优**，你可以自行维护人物及其头像。
- **视频合集**：可以把多个视频条目归入一个合集，合集成员有顺序，前端可以按顺序连续播放。

这些标签 / 人物 / 合集都可以在 app 里维护，也可以在导入时一次性带上（见下文）。

## 怎么导入

非 JAV 视频通过 `import-videos` 命令**就地索引**进来——注意是「索引」不是「搬运」：它只为每个视频文件建立一条记录，文件仍然留在原来的位置。

```bash
docker exec --user app -w /app sakuramedia python -m src.start.commands import-videos --source-path /mnt/volume1/media/clips
```

也可以只导入单个文件，并在导入时一并打标签、关联人物、归入合集：

```bash
docker exec --user app -w /app sakuramedia python -m src.start.commands import-videos --source-path /mnt/volume1/media/clips/one.mp4 --collection-id 1 --tag-ids 2,3 --person-ids 5
```

它和导入 JAV 影片（`import-media`）的关键区别：

- **就地索引，不搬运文件**，所以源路径不受「不能是媒体库目录」之类限制。
- 标题默认取文件名，**不抓取任何番号 / JavDB / DMM 元数据**。
- 已经按路径登记过的文件会自动跳过，可以放心重复执行。
- 缩略图不需要手动生成，会由 `aps generate-media-thumbnails` 任务自动补齐。

完整参数和示例见[常用命令 → 导入非 JAV 视频（无番号）](/guide/commands#导入非-jav-视频-无番号)。

## 适合什么场景

- 你有一批不属于 JAV 体系、也没有番号的视频，想纳入 SakuraMedia 统一浏览、打标签、做片段
- 你只需要「整理 + 播放」，不需要元数据抓取和自动下载

::: warning 非 JAV 视频不在「记番号、丢了再下」的范畴内
[收藏思路](/guide/collection-strategy)里「只记番号、不备份文件」那套，前提是 JAV 番号可以从网络重新下载回来。非 JAV 视频通常**没有番号、也无法再生**，一旦源文件丢失就找不回来——这类内容请按普通文件自己做好备份。
:::

# PornBox 视频管理

PornBox 是 SakuraMedia 用来纳管**没有番号、也没有外部元数据**的非 JAV 视频的板块——比如个人收藏、自己录制或剪辑、国产 / 国外资源等。这套体系和 JAV 影片完全平行，定位是「**仅播放 + 整理**」，不掺和订阅、下载这类 JAV 专属的自动化能力。

## 它和 JAV 影片有什么不同

| 维度 | JAV 影片 | PornBox 视频 |
|---|---|---|
| 核心标识 | 番号 | 无番号，标题默认取文件名 |
| 元数据来源 | JavDB / DMM 等外部抓取 | 无外部元数据，全部手动维护 |
| 组织维度 | 女优、标签、合集 | **仅视频合集**（不支持标签 / 人物）|
| 自动化能力 | 订阅、自动下载、推荐、相似影片、以图搜图 | **都不提供** |
| 播放能力 | 缩略图、进度、时刻、流播放 | **完全复用同一套** |
| 切片收藏 | 支持 | 同样支持（见[切片收藏](/guide/clips)）|

简单说：PornBox 视频借用了 SakuraMedia 的整套播放底座（缩略图、播放进度、精彩时刻、流播放），但不参与任何依赖番号 / 外部元数据的自动化链路。

## 怎么组织 PornBox 视频

PornBox 视频只通过**视频合集**来组织，不支持标签和人物维度。

- **视频合集**：可以把多个视频条目归入一个合集，合集成员有顺序，前端可以按顺序连续播放。

合集可以在 app 里维护，也可以在导入时一次性带上（见下文）。

## 怎么导入

PornBox 视频在 Web 界面的 **「媒体导入」页**（管理 → 媒体导入）导入——切到 **「PornBox 影片」** 标签：浏览后端白名单目录选源（一个目录或单个视频文件）→ 选「导入到媒体库」→ **必须选一个合集**（可现场新建）→ 选「导入方式」→ 开始导入。

导入在后台异步运行，可在该标签查看实时进度，并对失败文件「重导」。

它和 JAV 影片导入的关键区别：

- **不抓取任何番号 / JavDB / DMM 元数据**，标题默认取文件名。
- 文件落库语义与 JAV 一致：按「导入方式」**搬入媒体库**（`auto` 硬链接优先 / `cleanup-source` 复制后删源），不是「就地索引」。`cleanup-source` 同样不允许把媒体库目录作为源。
- 导入时**必须归入一个合集**（JAV 不需要）。
- 已按路径或内容指纹登记过的文件会自动跳过，可以放心重复导入。
- 缩略图不需要手动生成，会由 `aps generate-media-thumbnails` 任务自动补齐。

传输模式的详细行为见[常用命令 → 导入已有媒体](/guide/commands#导入已有媒体)。

## 适合什么场景

- 你有一批不属于 JAV 体系、也没有番号的视频，想纳入 SakuraMedia 统一浏览、整理成合集、做切片
- 你只需要「整理 + 播放」，不需要元数据抓取和自动下载

::: warning PornBox 视频不在「记番号、丢了再下」的范畴内
[收藏思路](/guide/collection-strategy)里「只记番号、不备份文件」那套，前提是 JAV 番号可以从网络重新下载回来。PornBox 视频通常**没有番号、也无法再生**，一旦源文件丢失就找不回来——这类内容请按普通文件自己做好备份。
:::

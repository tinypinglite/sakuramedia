# 115 网盘接入

SakuraMedia 已经把 115 网盘做成了「另一种媒体库后端」。除了原来的本地目录媒体库，你现在可以直接把 115 网盘当媒体库用——从 115 目录里导入影片、往 115 里做秒传归档、走 115 做离线下载、直接在 SakuraMedia 里播放 115 上的媒体。

::: warning 先看清楚它「不是什么」
- **不会替你上传视频**。它是把「115 上已经有的文件」纳管起来，或者「把本地文件通过秒传搬到 115」，不是一个 115 上传工具。
- **不是网盘客户端**。它只做媒体链路里 SakuraMedia 用得到的那部分（登录 / 目录浏览 / 秒传 / 离线下载 / 直链播放），不覆盖 115 的转存、社交、传输管理等能力。
:::

## 它能做什么

| 能力 | 说明 |
|---|---|
| 115 媒体库 | 扫码登录后建一个「115 网盘」类型的媒体库，等同于本地媒体库参与影片管理 |
| 从 115 目录导入 | 「媒体导入」页选 115 媒体库 → 浏览 115 目录 → 导入 JAV 或 PornBox 影片 |
| 秒传本地到 115 | 「媒体管理」页多选本地媒体 → 批量提交后台秒传批次到指定 115 网盘 |
| 115 离线下载 | 新增一种「115 离线下载」下载器，系统设置里下载偏好可选优先使用115 离线下载 |
| 直接播放 115 媒体 | 已入库的 115 影片可直接在 SakuraMedia 里通过302跳转直链播放，不用先下回本地，不依赖服务器带宽。 |


::: tip 离线下载依赖会员配额限制以 115 官方为准
SakuraMedia 只负责发起离线任务、秒传（媒体管理中仅支持秒传到115不支持普通上传）、目录浏览等调用，是否成功由 115 侧决定。
:::


## 使用前提


1. 115 网盘的会话是按「登录平台槽」分配的：**同一个平台槽只能有一处在线**，新登录会顶掉旧登录。SakuraMedia 会**持续占用**你选的那个登录平台槽（比如「支付宝小程序」），只要你在同一个平台上手动登录一次 115，就会把 SakuraMedia 顶下线、媒体库认证失效、后续任务全部报错. 
2. 115 网盘账号需要有VIP，且有离线下载配置，无VIP账号会被115限速， 无法正常播放生成缩略图等.



## 添加 115 网盘媒体库

进入「设置 → 媒体库 → 新增媒体库」，在后端类型里选「115 网盘」，然后走一段扫码登录流程：

1. **命名**：起一个便于识别的媒体库名，例如 `115 主账号`。
2. **选登录平台**：目前放开两个选项——
   - **支付宝小程序**（推荐）
   - **微信小程序**

   SakuraMedia 会把这个平台槽长期占用。**必须勾选下方的风险确认**表示你知道会占用这个槽，才能继续。
3. **扫码**：用 115 手机 App 打开对应平台的登录入口扫码；等扫完、在手机上确认，SakuraMedia 就会自动完成媒体库创建并保存登录凭据。
4. 出现「115 媒体库已创建」提示后，回到媒体库列表就能看到新条目，副标题会显示 `115 网盘 · 支付宝小程序`（或你选的平台）。


## 用 115 做离线下载

进入「设置 → 下载器 → 新增下载器」，类型选「115 离线下载」，只需要：

- **下载器名称**：便于识别即可
- **目标媒体库**：选一个 115 网盘媒体库， 此后通过此下载器提交的离线下载任务都会归入这个媒体库，并离线到对应的115账号中。


绑定索引器：

- **BT（公网）索引器**可以绑 115 离线下载器（一条索引器可同时绑多个下载器，SakuraMedia 会按你在索引器里的绑定关系分发任务）。
- **PT（私有）索引器不能绑 115 离线下载器**。大部分PT站都不会让你把种子提交给115离线下载，这里只是为了避免误操作。 如果真想用115离线 PT种子， 你可以把PT索引器改成BT类型再绑定。

日常使用中要注意的差异：

- **115 离线任务不能暂停 / 恢复**。下载列表里的「暂停 / 继续」按钮只对 qBittorrent 任务生效。
- **配额是 115 侧管的**。返回「本月离线下载配额已用尽」时，只能等次月或升级 115 会员。
- **同一资源已存在于 115 的非托管目录时会拒绝接管**，会话提示 `该资源已存在于 115 的非托管目录，无法接管`——这时候如果确实想让 SakuraMedia 管这个文件，需要你自己在 115 里挪出那个目录，或者干脆走「从 115 目录导入」把它纳管起来。
- 删除任务时勾选「同时删除 115 中已下载的文件」会把文件放进 115 回收站；不勾则只解除 SakuraMedia 侧的关联。**已经导入媒体库的文件不受影响**（因为已经被复制/搬到媒体库目录了）。



## 从 115 目录导入影片

也可以把 115 上已经有的影片纳管进 SakuraMedia。入口在**「管理 → 媒体导入」**页：

- **导 JAV 影片**：切到「JAV 影片」标签，目标媒体库选你刚建的 115 媒体库，来源选「115 网盘目录」，浏览到目标 CID 目录，选好后开始导入。
- **导 PornBox 视频**：切到「PornBox 影片」标签，来源选择同理支持 115 目录，导入时**必须归入一个视频合集**，其余行为和本地 PornBox 导入一致（详见 [PornBox 视频管理](/guide/videos)）。

**导入方式**（`TransferMode`）在 115 场景下和本地略有不同：

| 模式 | 本地媒体库 | 115 媒体库 |
|---|---|---|
| 硬链接优先（保留源文件） | ✅ 默认 | ❌ 不适用 |
| 复制并保留源文件 | ❌ | ✅ 默认（在 115 内部复制到媒体库目录，源目录不变）注意这会占用双倍空间!|
| 复制后删除源文件 | ✅ | ✅（在 115 内部把源目录里的文件搬走）|

::: tip 从 115 导入不会占用本地存储
115 目录导入的整个过程都在 115 服务器内部完成，SakuraMedia 只是发起复制/搬运指令，然后拉取元数据和字幕。**下载带宽和本地磁盘都不会因为一次 115 导入而被占满**。

注意：如果遇到字幕文件，是会将字幕文件下载到本地的。

:::

导入过程中的常见失败原因（会在「媒体导入」页的失败列表里显示）：

- `115 已封禁：文件被 115 标记违规` —— 拿不到直链，无法入库
- `115 搬运失败` —— 云端复制或对账阶段失败，通常可以重导
- `115 单文件改名未成功` —— 目标名和实际名不一致，也可以重导
- `字幕下载失败` —— 影片本体已入库，仅字幕告警，不阻塞

## 把本地媒体秒传到 115

如果你想把已经在本地的影片归档到 115（腾出本地盘），可以走**「管理 → 媒体管理」**页的秒传功能：

1. 顶部 tab 切到「媒体列表」。
2. 用筛选找到要秒传的**本地媒体**（云端媒体没意义），多选。
3. 点「秒传到 115」，弹窗里选一个目标 115 媒体库（如果只有一个会默认选中），点「开始秒传」。
4. 后端会创建一个秒传批次，切到「秒传批次」tab 就能看到进度；批次在进行中时会自动刷新。
5. 成功的条目会**自动切换到云端存储并删除本地文件**；失败的条目保留在本地，可以在批次卡上「重试失败项」。

::: warning 秒传能不能成功，取决于 115 云端有没有这份文件的哈希
「秒传」本质是用文件哈希去 115 云端匹配，命中才能瞬间完成上传；命不中的文件会失败——SakuraMedia 不会替你走「真上传」链路把文件推上去。所以对**较冷门 / 自己剪辑 / 稀有资源**的秒传成功率会明显低于主流 JAV 影片，这是 115 秒传本身的限制。
:::



## 播放 115 上的影片

已入库到 115 媒体库的影片，在详情页和播放器里的操作和本地媒体基本一致——SakuraMedia 会向 115 请求直链再交给播放器。你不用先把文件下回本地。

删除时会有额外提示：**115 媒体的删除会把媒体文件送入 115 回收站**，同时抹掉 SakuraMedia 里的记录。这一步不可撤销，只能去 115 回收站里找回。

## 认证失效怎么处理

因为 SakuraMedia 占的是「登录平台槽」，只要发生下面几种情况都可能让认证失效：

- 你（或另一个用同一账号的人）在**同一个平台**上重新扫码登录了 115
- 你在 115 App 里主动把这个登录设备踢下线
- 长时间不用被 115 侧超时清理

重新认证：

1. 进入「设置 → 媒体库」。
2. 找到失效的那个 115 媒体库条目，点条目右侧的**刷新图标**（🔄）。
3. 弹出一样的扫码框，选好登录平台（默认沿用上一次的选择），扫码确认。
4. 成功后 SakuraMedia 会更新登录凭据，之前的媒体库 / 下载器 / 已入库文件都不用重建，会话恢复即可继续用。
